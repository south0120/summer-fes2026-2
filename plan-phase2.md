## 実装スペック（確定方針ベース）

### A. 参加型ヒーロー（承認制）
- **DB**: `hero_submissions` テーブル（postersと同型）。列: id / user_id / kind('background'|'logo') / image_path / approved bool(default false) / created_at。RLS = public read は `approved=true` のみ / insert は本人 / update(承認)は service_role のみ。
- **Storage**: バケット `hero`（public read）。パス `${user.id}/${uuid}.${ext}`。
- **投稿UI**: `/hero/new`（背景 or ロゴを選んで画像アップ）。ログイン必須（既存auth流用）。投稿直後は承認待ち＝トップ未反映。
- **承認導線**: admin用エンドポイント `/api/admin/hero/approve`（service_role・自分だけ）。当面は簡易でOK（一覧＋承認ボタン、または直SQL/管理ページ）。
- **表示**: `Hero.tsx` の bg画像・ロゴ差し込みを動的化。approved=true の中からランダム1枚（背景）＋ランダム1枚（ロゴ、無ければ公式`title-logo.webp`にfallback）。SSR or クライアントで選択。
- ⚠️公式ロゴ(`public/art/title-logo.webp`)は承認済ロゴが無い時のfallbackとして常に残す。

### B. ミニゲーム5つ（非公開ページ）
- 案: ヨーヨー釣り(タイミング)/もぐらたたき(反射連打)/型抜き(精密)/太鼓リズム(音ゲー)/福引ガチャ(運)。
- **非公開ページ**: 推測されにくいパス `/preview-games`（トップ・ヘッダーNAVから未リンク・sitemap/robots対象外）にまとめて置く。各ゲームは既存パターン踏襲: `src/app/<game>/page.tsx` + `src/components/<game>/<Game>.tsx` + GameShell/palette.ts、canvas実装。
- リーダーボードは `scores.game` 列＋RankingPanelで即対応可（GameKeyに値追加）。ただし非公開段階ではボード接続は任意。
